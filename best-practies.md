# Senior Android Best Practices (2025)

- Always use multi-module architecture with feature modules
- Never put business logic in View or Composable
- Use Baseline Profiles + Macrobenchmark in every release
- Write UI tests with Compose Testing + Turbine for ViewModels
- Keep app size under 10MB after full R8 optimization
- Use Ktor instead of Retrofit for new projects (KMP ready)
- Design System must be published as internal library
- Every screen must support dark mode, dynamic colors, and large fonts
